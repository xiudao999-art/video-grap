# 金币 / 打赏 / 积分 UI 视觉模式研究报告

## 一、研究方法

本报告基于 Deep Research adversarial verification 流程，对音乐直播平台、游戏娱乐平台、直播打赏系统中的金币/积分/礼物 UI 视觉模式进行了专门调研。共保留 **25 条经过验证的设计趋势 claim**。

---

## 二、核心发现

### 1. 虚拟货币图标形态

#### 1.1 宝石/钻石形态
- **Twitch Bits**：使用**宝石图标（gem icon）** 作为虚拟货币入口，位于聊天窗口底部
- 用户点击 "Bits gem icon" 打开 "gem Cheermotes menu"
- 视觉隐喻：宝石 = 珍贵、打赏、支持

#### 1.2 硬币/金币形态
- **3D 硬币图标** 推荐用于：
  - Hero 区块
  -  onboarding 引导页
  -  幻灯片/社交图形
  -  功能展示区块
- **简单 SVG/PNG/矢量图标** 推荐用于：
  - 表格、菜单、token 列表
  - 紧凑界面控件
- 设计原则：空间越大，可用 3D/玻璃质感；空间越小，用扁平/线性图标

### 2. 虚拟礼物分类体系

根据某直播平台礼物市场分类，虚拟礼物/特效可分为 10 大类：

| 类别 | 用途 |
|------|------|
| **Gift** | 普通礼物 |
| **Car** | 进场座驾/豪车特效 |
| **Entry effect** | 入场动画 |
| **Avatar frame** | 头像框 |
| **Medal** | 勋章/徽章 |
| **Chat bubble** | 聊天气泡 |
| **Home page effects** | 主页特效 |
| **Data Frame** | 数据框/信息卡 |
| **Background** | 背景特效 |
| **Emoji** | 表情礼物 |

### 3. 互动礼物标识

- **YouTube Gifts**：用**紫色 sparkle（闪光）图标** 标记互动礼物
- 位置：礼物选择器（gift picker）中
- 作用：帮助观众识别增强型/互动型礼物选项

### 4. 打赏金额与动效层级

- **Streamlabs Alert Box** 支持根据打赏金额设置不同层级的 alert：
  - `$5 小费` → 普通 alert
  - `$50 小费` → 巨大、华丽的 alert
- 设计原则：**打赏金额越高，alert 越华丽、越显眼**
- 可设置条件："Donation amount is at least <amount>", "Donation amount is the largest this stream"

### 5. Alert 动画系统

Streamlabs 的 alert 系统支持三类动画：
1. **Alert 进入动画**（entrance animation）
2. **Alert 退出动画**（exit animation）
3. **文字在屏动画**（text animation while onscreen）

### 6. 忠诚度徽章系统

- **Twitch Subscriber Tiers**：Tier 1 / Tier 2 / Tier 3 订阅等级徽章
- 功能：
  - 显示在聊天用户名旁边
  - 订阅时间越长，徽章越闪亮
  - 作为数字身份标识和地位象征
- 设计要点：小尺寸、高辨识度、随时间/等级升级变化

### 7. 礼物价值与视觉反馈

- **抖音直播礼物经济**：礼物价值越高，屏幕动画越宏大
- 设计逻辑：
  - 低价礼物 → 简单图标/小动画
  - 高价礼物 → 全屏特效、座驾进场、粒子雨
  - 目的：增加发送者的可见度和社交地位

### 8. 兑换/收益可视化

- **YouTube Jewels/Rubies**：
  - 2 Jewels = 1 Ruby = $0.01
  - 收益计算可视化，让主播清晰了解打赏价值
- 设计要点：
  - 明确展示货币兑换关系
  - 用进度条/数字展示累计收益

## 三、视觉设计规范建议

### 1. 金币/积分图标

| 场景 | 推荐形态 | 材质 |
|------|---------|------|
| Hero 区/首屏 | 3D 旋转金币、发光 token | 金属质感、渐变、高光 |
| 导航/菜单 | 简单金币 SVG 图标 | 扁平/线性 |
| 列表/表格 | 小型金币 icon | 单色/双色调 |
| 成就徽章 | 3D 徽章/奖牌 | 金属+珐琅质感 |

### 2. 礼物视觉层级

| 礼物价值 | 视觉处理 |
|---------|---------|
| 低价值（1-10 币） | 简单 2D 图标，小动画 |
| 中价值（10-1000 币） | 3D 图标，中等动画，音效 |
| 高价值（1000+ 币） | 全屏特效、粒子雨、进场动画、座驾 |
| 顶级（火箭/宇宙等） | 全屏霸屏、房间广播、特殊音效 |

### 3. 配色方案

| 元素 | 推荐色 |
|------|--------|
| 金币/积分 | 金色 `#FFD700`、金黄色 `#FFA500`、铜色 `#B87333` |
| 宝石/钻石 | 蓝色 `#00D4FF`、紫色 `#B026FF`、粉色 `#FF2D7A` |
| 高价值礼物 | 彩虹渐变、金色粒子、红色火焰 |
| 背景 | 深色底（黑/深紫/深蓝）突出金币发光效果 |

### 4. 布局组件

必备组件：
1. **金币余额显示** — 顶部导航或侧边栏
2. **礼物选择器** — 底部/侧边网格，分类筛选
3. **打赏按钮** — 高对比 CTA
4. **Alert 区域** — 屏幕中央/顶部，显示打赏信息
5. **排行榜/榜单** — 财富榜、贡献榜
6. **进度条/成就** — 等级进度、每日任务
7. **特效预览** — 礼物 hover 时显示预览

### 5. 动效规范

| 动效类型 | 用途 | 注意 |
|---------|------|------|
| 金币掉落 | 获得奖励/充值成功 | 不要过度，避免视觉噪音 |
| 礼物飞入 | 用户送出礼物 | 从送出者位置飞向主播 |
| 全屏特效 | 高价值礼物 | 控制频率，避免打扰 |
| 数字滚动 | 积分增加 | 增强获得感 |
| 徽章升级 | 等级提升 | 闪光+缩放+粒子 |
| 进度条填充 | 任务完成 | 配合奖励弹窗 |

## 五、针对「金币+美女+歌曲」主题的应用建议

### 音乐/直播平台
1. **顶部导航**：金币余额 + 充值按钮（金色高亮）
2. **主播卡片**：圆形头像带光晕边框 + 等级徽章
3. **礼物栏**：底部/右侧礼物选择器，紫色 sparkle 标记互动礼物
4. **Alert 区**：屏幕中央显示 "XXX 送出 火箭 ×10" + 全屏特效
5. **歌曲互动**：点歌需消耗金币，进度条显示点歌热度

### 游戏/娱乐平台
1. **角色/IP 展示**：3D 角色立绘 + 金币/宝石装饰
2. **奖励系统**：任务完成 → 金币掉落 + 成就徽章
3. **抽奖/开箱**：宝箱开启动效 + 金币飞溅
4. **等级体系**：VIP 徽章、头像框、进场特效

## 六、设计注意事项

1. **避免赌博感**：金币设计要偏向"游戏奖励/打赏礼物"，避免像赌场筹码
2. **控制动效密度**：全屏特效频繁会打扰核心内容（直播/音乐）
3. **清晰的价值感**：展示金币兑换关系，让用户理解打赏价值
4. **可访问性**：高对比度确保深色模式下金币/文字可读
5. **移动端适配**：礼物选择器、排行榜等组件需适配小屏幕
